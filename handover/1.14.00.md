## 3. I2C Display Implementation

### 3.1 Summary
- **Library:** `greiman/SSD1306Ascii @ ^1.3.0` (lightweight text-only OLED, ~2KB flash)
- **Supported types:** SSD1306 OLED (0x3C/0x3D), SH1106 OLED (0x3C/0x3D)
- **Files:**
  - `include/display_driver.h` — `DisplayDriver` class wrapping SSD1306Ascii with i2cMutex
  - `include/config.h` — `display_enabled`, `display_i2c_addr`, `display_brightness` fields
  - `include/web_pages.h` — Display section in Network tab (type, address, brightness)
  - `src/main.cpp` — global `display`, init in `setup()`, update ~2Hz in `networkTask`
- **Display lines (4-line OLED):**
  1. `IP 192.168.1.93`
  2. `ETH Link Up` / `Wi-Fi Connected` / `No Link`
  3. `U 0-2` (universe range from active output channels)
  4. `FPS 40  Heap 57K`

### 3.2 Freeze Protection
- **I2C probe ก่อน init:** `Wire.beginTransmission(addr); Wire.endTransmission()` — ถ้าไม่ ACK, `begin()` return false ทันที (ไม่รอ library timeout)
- **Heartbeat probe ก่อนทุก update:** เช็ค I2C address ทุก 500ms ก่อนเรียก library functions
- **Auto-disable:** ถ้า I2C error ติดต่อกัน >= 3 ครั้ง → `_active=false`, delete OLED object, free RAM
- **Every method guarded:** `setBrightness()`, `end()` ล้วน probe ก่อน execute

## 4. ZC Pin Safety

### 4.1 Code-Level Guard
- `dimmer_control.h:65` — ถ้า `sysCfg.zc_pin == 255`, `DimmerControl::begin()` return ทันที ไม่เปิด timer หรือ attach interrupt
- ถ้าตั้ง ZC pin เป็นค่าใดค่าหนึ่งแต่ไม่ได้ต่อสาย → interrupt storm ได้ (hardware issue, ต้องตั้งเป็น 255 ถ้าไม่ใช้)

### 4.2 Web UI Warning
- Outputs tab มี `<div id="zc-warning">` สีแดง
- `renderOutputs()` ตรวจว่า `outputs` มี AC Dimmer (type 3) และ `zc_pin` = 255 หรือไม่
- ถ้าใช่ → แสดง warning: "AC Dimmer output(s) but ZC pin is Disabled"
- Warning หายอัตโนมัติเมื่อลบ AC Dimmer หรือตั้ง ZC pin

## 6. Known Issues

### 6.1 Pin rows ไม่เปลี่ยนตาม type
- **Status:** ✅ Fixed — `renderPinRows()` ใช้ `mkSel()` สร้าง HTML โดยตรง ไม่พึ่ง DOM elements ที่มีอยู่ก่อน

### 6.2 ZC Pin ไม่ได้ต่อ (hardware risk)
- ถ้าตั้ง ZC pin ≠ 255 แต่ไม่ต่อสายจริง → interrupt storm → controller freeze
- **Workaround:** ตั้ง ZC pin เป็น Disabled (255) ถ้าไม่มี AC Dimmer

### 6.3 Display ยังไม่ได้ต่อจริง
- Display code มี freeze protection อยู่แล้ว (I2C probe + auto-disable)
- Config default: Disabled (display_enabled=0)
- เปิดใช้ที่ Network tab → I2C Display → เลือก SSD1306/SH1106 + set address

## 7. Dynamic Output Scoring System

### 7.1 Summary
- **Purpose:** ป้องกันการกำหนด output เกินขีดจำกัด hardware ของ ESP32 โดยให้คะแนนแต่ละ output type ก่อนบันทึก ถ้าเกิน 100 คะแนน → block
- **Files:**
  - `include/scoring.h` — **New** — `scoreForType()`, `outputChannelScore()`, `totalOutputScore()`, `totalOutputScoreFromJson()`, `getOutputTypeName()`
  - `src/main.cpp` — `GET /api/scoring` (total/limit/remaining/per-channel breakdown), validation ใน `POST /api/outputs`
  - `include/web_pages.h` — Score bar (สีเขียว ≤80 / เหลือง 80-100 / แดง >100), per-channel `<span class="pill">`, `calculateScore()` + `updateScoreBar()`, `checkScoreAndWarn()` ใน `addOrUpdateOutput()`, ปุ่ม Save ถูก disable ถ้าเกิน 100

### 7.2 Scoring Matrix

| Output Type | Base Score | Variable |
|:---|---:|---:|
| LED Strip (type 0) | 2.0 | +1.0 ต่อ led_count/100 px |
| DMX (type 1) | 4.0 | — |
| Relay / Solenoid (type 2, 9) | 0.5 | — |
| AC Dimmer / Servo / 7-Seg / Analog LED (types 3, 5, 6, 8, 10, 13, 14) | 2.0 | — |
| DC Motor (type 7) | 8.0 | — |
| Buzzer / Smoke Machine (type 11, 12) | 1.5 | — |

- **Limit:** 100.0 (hard ceiling, block save)
- **Precision:** float, แสดง 1 decimal ใน UI

### 7.3 Validation Flow
1. User clicks Save Channels → `addOrUpdateOutput()` calculates `totalScore(newOutputs)`
2. ถ้า > 100 → `alert()` + return (ไม่ให้ save)
3. ถ้า ≤ 100 → ผ่าน, run resource warnings → confirm → save
4. Server-side validation ใน `POST /api/outputs` — ถ้า score > 100 → HTTP 400
5. UI score bar auto-update via `updateScoreBar()` ทุกครั้งที่ `renderOutputs()` เรียก
