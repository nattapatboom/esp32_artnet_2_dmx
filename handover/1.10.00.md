## Carried Forward from v1.9.0

### Output and Protocol Runtime
- Art-Net, sACN, and ESP-NOW receive paths now share `OutputControl::mapDmxDataToChannels()`.
- sACN now renders through the same output path as Art-Net by setting `networkFramePending`.
- Art-Net and sACN UDP ports are configurable from the web UI, with standard defaults kept for normal setups.
- Outputs can now be manually tested from the web UI without incoming Art-Net/sACN/ESP-NOW data.
- ESP-NOW DMX forwarding is fragmented into 200-byte chunks with `offset` and `totalLength`.
- ArtPollReply universe discovery includes all configured output types.

### Output Addressing
- `OutputChannel::start_address` supports small output devices sharing the same universe.
- LED strips remain consecutive full-universe consumers starting at channel 1.
- DMX Serial remains a full 512-channel universe output.
- Small output examples:
  - Relay at `U0:CH1`
  - Dimmer at `U0:CH2`
  - WiZ RGB at `U0:CH10`
  - Stepper 16-bit at `U0:CH20`

### Output Types Covered
- LED Strip
- DMX Serial
- Relay
- AC Dimmer
- WiZ Smart Bulb
- DC PWM Dimmer
- DC Motor
- Stepper Motor
- RC Servo
- Solenoid Trigger

### Removed Standalone/Failsafe Scope
- Standalone/Clock, Failsafe Automation, Schedule/NTP, and Custom Sequence programming have been removed from the active firmware.
- Reason: the feature set was not yet suitable for the required program-like output control and was increasing flash/OTA pressure.
- If local show playback is needed later, it should be redesigned as a separate, capacity-checked feature instead of restoring the old built-in effects.
