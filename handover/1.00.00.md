# Handover เอกสารส่งมอบงาน v1.0.0: Art-Net Node Device (WT32-ETH01)

ไฟล์เวอร์ชันนี้เป็นโครงร่างและการติดตั้งระบบซอฟต์แวร์เฟิร์มแวร์หลักของอุปกรณ์แปลงสัญญาณจาก **Art-Net (ผ่านสาย LAN/Ethernet)** ไปยังการขับสัญญาณ DMX512 และ Pixel LED รวมถึงความสามารถในการคุมหลอดไฟ WiZ และการทรานส์ซีฟเวอร์ข้อมูลไร้สายด้วยโปรโตคอล ESP-NOW

---

## 1. ฟีเจอร์ที่เสร็จสมบูรณ์ในเวอร์ชัน v1.0.0 (Completed Features)

* **ระบบเครือข่ายและการแลกเปลี่ยนข้อมูล (Network Core)**:
  * จัดการการ์ด LAN8720A บนบอร์ด WT32-ETH01 รองรับทั้ง DHCP และการกำหนด IP แบบคงที่ (Static IP)
  * เขียนระบบค้นหาตัวตน **ArtPoll / ArtPollReply** ทำให้อุปกรณ์สามารถแสดงตัวตนบนเครือข่ายสำหรับซอฟต์แวร์ควบคุม (TouchDesigner, Resolume, QLC+, MA2/MA3) โดยอัตโนมัติ
  * ระบบปล่อยสัญญาณ Wi-Fi SoftAP ในตัวเองสำหรับการเชื่อมต่อหน้าเว็บตั้งค่าผ่านคอมพิวเตอร์และมือถือ มีการใช้ mDNS โดเมนเข้าที่หน้า `http://artnet.local`
  * โหมดสแตนด์บายของเครือข่าย (AP Timeout): สำหรับโหมด ESP-NOW Slave จะเปิดใช้งานสัญญาณ Wi-Fi AP สำหรับตั้งค่าในช่วงบูตแรก 2 นาที หลังจากนั้นหากไม่มีการเชื่อมต่อ AP จะปิดตัวเองโดยอัตโนมัติเพื่อป้องกันสัญญาณรบกวนในระหว่างจัดงานจริง

* **โหมดส่งผ่านข้อมูลไร้สาย (ESP-NOW Wireless Bridge)**:
  * พัฒนาระบบส่งสัญญาณแบบ Master-Slave ระหว่างบอร์ด ESP32 เพื่อให้สัญญาณ Art-Net สามารถข้ามฝั่งวิทยุไร้สายโดยไม่ต้องผ่านเราเตอร์
  * โหมด Master (ส่ง): รับข้อมูลจาก LAN แล้วส่งต่อ DMX แพ็กเกจผ่าน ESP-NOW
  * โหมด Slave (รับ): รับข้อมูลจากวิทยุ แล้วนำไปแปลงออกทางบอร์ดเอาต์พุต

* **ตัวขับสัญญาณโคมไฟและไฟเส้น (Output Drivers)**:
  * โหมด DMX512 (ส่งออกอย่างเดียว): ใช้ไลบรารี `esp_dmx` ในการขับสัญญาณผ่าน Hardware UART ขา TX2 (GPIO 17) ปรับความเร็วกระพริบส่งออก (DMX FPS) ได้สูงสุด 44 FPS
  * โหมด Pixel LED: ขับไฟเส้น WS2812B/UCS1903 ด้วย `NeoPixelBus` (ใช้ RMT Channel 0 ปราศจากการรบกวนสัญญาณจากระบบเครือข่าย)
  * การสลับจัดเรียงแม่สีด้วยซอฟต์แวร์ (Dynamic Color Reordering): ให้ผู้ใช้สามารถเลือกสลับโทนสีไฟ (GRB, RGB, BRG, RBG) ได้จากหน้าเว็บ ไม่จำเป็นต้องแก้โค้ดและเบิร์นใหม่
  * ระบบจัดส่งช่องสัญญาณข้ามจักรวาล (Multi-Universe Mapping): สามารถขับไฟเส้นได้ยาวข้ามจักรวาลมากกว่า 170 พิกเซลขึ้นไป โดยระบบจะดึง DMX ของ Universe ถัดๆ ไปมาประมวลผลต่อกันโดยอัตโนมัติ (รองรับสูงสุด 8 Universes หรือ 1024 พิกเซล)

* **โหมดคุมหลอดไฟ WiZ Bulbs (WiZ Local UDP Control)**:
  * แปลงแพ็กเก็ต DMX ส่งคำสั่งควบคุมหลอดไฟ WiZ (รุ่น 8.5W) ผ่าน JSON UDP พอร์ต 38899
  * ระบบจำกัดความเร็วการส่งข้อมูล (Rate Limiting) ที่ 10Hz เพื่อป้องกันไม่ให้ชิปของหลอดไฟแฮงก์
  * การควบคุมแบบ 3 ช่องสัญญาณ (RGB Only - Auto Dimmer): เฟิร์มแวร์จะคำนวณระดับหรี่แสงอัตโนมัติ ทำให้ผู้ใช้ระบุชนิดโคมไฟเป็น Generic RGB ทั่วไปในโปรแกรมคุมไฟได้ทันที
  * การควบคุมแบบ 4 ช่องสัญญาณ (RGB + Dimmer): ควบคุมสี R, G, B และ Dimmer แยกเป็นอิสระ

* **หน้าเว็บปรับแต่งค่าอัจฉริยะ (Responsive Web Configuration UI)**:
  * ออกแบบหน้าเว็บด้วยรูปแบบ Glassmorphism และรองรับการเปลี่ยนมุมมองอัตโนมัติทั้งมือถือและคอมพิวเตอร์ (Responsive layout)
  * ฝังโลโก้บริษัท `[CHAL]_128_64.bmp` ที่ผ่านการบีบอัดเป็น 1-bit PNG ขนาด 266 ไบต์ และเข้ารหัส Base64 (0 RAM Overhead) ทำให้หน้าเว็บโหลดรวดเร็วที่สุด
  * หน้ารับ-ส่งค่าสถานะเรียลไทม์ (Live Telemetry AJAX) เช่น แสดงสถานะว่าได้รับ Art-Net กี่แพ็กเก็ต, ค่าเวลาจริงจาก NTP, ไอพีปัจจุบัน
  * Visual Custom Sequence Builder: หน้าเว็บสำหรับเพิ่ม/แก้ไขขั้นตอนไฟสแตนด์อโลน (Fade/Hold Time, Color Picker) และอัปโหลดไปบันทึกเป็น JSON ลงใน LittleFS
  * ระบบอัปเกรดแบบไร้สาย (OTA Firmware Web Update)

## 3. ขั้นตอนการนำไปทดสอบ (Verification & Testing Guide)

### 3.1 การเชื่อมต่อฮาร์ดแวร์พื้นฐาน
1. ต่อขารับข้อมูลไดรฟ์เวอร์ไฟพิกเซลเข้ากับขา GPIO 4 (หรือพินอื่นที่เลือกบนเว็บ)
2. ต่อชิปแปลงระดับแรงดัน SN74HCT125N คั่นสัญญาณควบคุมเอาต์พุต โดยเฉพาะขา GPIO 12 ป้องกันแรงดันกะพริบตอนบูต
3. เชื่อมต่อตัวส่งสัญญาณ MAX485 เข้ากับพิน GPIO 17 (สำหรับเอาต์พุต DMX)
4. จ่ายไฟเลี้ยง 5V/12V เข้ากับบอร์ดไฟพิกเซล พร้อมติดตั้งฟิวส์ป้องกันสายไหม้ และทำการจ่ายไฟเลี้ยงเพิ่ม (Power Injection) ทุกๆ 150-200 ดวง

### 3.2 ขั้นตอนการตรวจสอบซอฟต์แวร์
1. **ทดสอบเปิดเครื่องและ SoftAP**:
   - บูตตัวบอร์ดโดยยังไม่ต้องเสียบสาย LAN
   - ค้นหาสัญญาณ Wi-Fi ชื่อ `ESP32-ArtNet-Setup` เชื่อมต่อโดยไม่ต้องป้อนรหัสผ่าน (เป็นค่าเริ่มต้น)
   - เปิดบราวเซอร์พิมพ์คำว่า `http://artnet.local` หรือ `192.168.4.1` เพื่อเข้าสู่หน้าเว็บตั้งค่า
   - ทดสอบว่าไฟ LED3 กะพริบช้าๆ (1Hz) แสดงสถานะ SoftAP
2. **ทดสอบบันทึกและจำค่าในแฟลช**:
   - ป้อนชื่อ SSID/รหัสผ่านและค่าพิน หรือเปลี่ยนจำนวนพิกเซลไฟบนหน้าเว็บ กดปุ่ม Save Settings
   - อุปกรณ์จะประมวลผลเซฟและรีสตาร์ตตัวเองภายใน 2 วินาที สังเกตว่าค่าตั้งค่าใหม่จะถูกดึงมาใช้อัตโนมัติหลังบูต
3. **ทดสอบใช้งานร่วมกับ TouchDesigner (DMX Out CHOP)**:
   - เสียบสาย LAN เข้ากับบอร์ด ESP32 และเชื่อมเข้าวงเน็ตเวิร์กเดียวกันกับคอมพิวเตอร์
   - ในโปรแกรม TouchDesigner เลือกสร้างโหนด **DMX Out CHOP**
   - ตั้งค่า Protocol เป็น **Art-Net**, ตั้งค่า IP address เป็น IP ของตัวบอร์ด ESP32 (Unicast) หรือบรอดแคสต์ และตรง Universe ตั้งให้ตรงกับ Universe ที่ระบุในเครื่อง (ปกติคือ 0)
   - ป้อนค่าสี R, G, B สังเกตการแสดงผลบนไฟเส้นพิกเซลว่าขยับตามแบบเรียลไทม์ และตรวจดูไฟกะพริบข้อมูล (LED4 / GPIO 17) บนตัวบอร์ด
4. **ทดสอบตัวควบคุมหลอดไฟ WiZ**:
   - เข้าหน้าเว็บตั้งค่า ติ๊กถูกที่ช่อง Enable WiZ Bulb Control
   - ระบุ IP ปลายทางของหลอดไฟ WiZ หรือบรอดแคสต์ไปที่ `255.255.255.255`
   - ป้อนช่องดีเอ็มเอ็กซ์เริ่มต้น (DMX Start Channel) ของหลอดไฟ WiZ (เช่น ช่อง 100)
   - ส่งคำสั่งความสว่างและสีผ่าน TouchDesigner จาก DMX แชนเนลที่ 100, 101, 102 สังเกตหลอดไฟปรับโทนตามความสว่างอย่างถูกต้อง

## 4. แผนการพัฒนาต่อไป (Next Steps)
1. **การปรับแต่งใน PCB**: ออกแบบบอร์ด PCB Shield สำหรับ WT32-ETH01 เพื่อเชื่อมโครงสร้างขั้วต่อต่อสายไฟให้แข็งแรงทนทานในการติดตั้งถาวรในร้านอาหารหรือบาร์
2. **3D Printed Box**: ออกแบบกล่องพลาสติกขนาดกะทัดรัดด้วยเครื่องพิมพ์ 3 มิติ เพื่อใช้ป้องกันบอร์ดช็อตกับฝุ่นหรือโครงสร้างโลหะรอบข้าง
3. **การทดสอบความเสถียรต่อเนื่อง (Long-term Stress Test)**: รันการส่งข้อมูลแสงความละเอียดสูงผ่านเครือข่ายเป็นเวลาอย่างน้อย 48 ชั่วโมงต่อเนื่อง เพื่อวิเคราะห์การรักษาความเสถียรของอุณหภูมิและการจัดการหน่วยความจำบนบอร์ด ESP32
