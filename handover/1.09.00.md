## Changes Made in v1.9.0
1. **Shared DMX Mapping**
   - Added `OutputControl::getUniverseCount()` and `OutputControl::mapDmxDataToChannels()`.
   - Art-Net, sACN, and ESP-NOW receive paths now share the same channel mapping behavior.
   - RGBW LED universe calculation now uses 128 pixels/universe instead of RGB-only 170 pixels/universe.
   - A single LED output GPIO can span consecutive universes based on LED count. Example: RGB 500 LEDs uses 1,500 channels, so it consumes 3 universes starting at `start_universe`.
2. **Network Render Trigger**
   - Added global `networkFramePending`.
   - `outputTask` now renders network frames from Art-Net or sACN, and updates WiZ targets before LED/motion rendering.
   - ESP-NOW slave receive path now refreshes WiZ targets too.
3. **sACN Cleanup**
   - Rewrote `include/sacn_control.h` in ASCII to remove mojibake comments and simplify maintenance.
   - Removed unsafe fallback universe expression.
4. **Solenoid Persistence**
   - Added safe defaults for solenoid runtime/config fields.
   - Added JSON load/save for `solenoid_mode`, `solenoid_threshold`, `solenoid_pulse_ms`, `solenoid_pre_delay`, and `solenoid_post_delay`.
   - Web UI now loads/saves these fields and shows the main GPIO selector for Solenoid outputs.
5. **Chunked POST Handling**
   - Added request-body accumulation for AsyncWebServer body callbacks.
   - Updated `/api/settings`, `/api/outputs`, `/api/sequence`, and `/api/espnow-peers`.
6. **Safe Settings String Copy**
   - Added `copyConfigString()` and replaced settings POST `strncpy` calls.
7. **ESP-NOW DMX Fragmentation**
   - Added `ESPNOW_DMX_CHUNK_SIZE` and split DMX universe forwarding into 200-byte chunks.
   - Added packet `offset` and `totalLength` fields so receivers can place partial DMX data back into the correct channel range.
   - ESP-NOW receiver now maps chunks through the shared DMX mapper with offsets.
8. **Custom Sequence Format Fix**
   - Web UI now edits/saves `fade_ms` and `hold_ms`, matching firmware.
   - Firmware still accepts legacy `ms` fields as a fallback for old saved sequence files.
9. **Standalone/Failsafe Output Coverage**
   - Fixed Solid Color first-render bug when configured color is already `255,255,255`.
   - Added RGB/RGBW-aware `writeLedPixel()` helper for standalone LED effects.
   - Solid/Color/Fade/Strobe-style standalone output now writes a single intensity DMX value to non-color output types such as Relay, Dimmer, Motion, and Solenoid.
10. **DMX RMT UI Access**
   - Removed the Web UI limit that blocked adding more than two DMX outputs.
   - UI now documents that first two DMX outputs use UART and additional DMX outputs use RMT fallback.
11. **GPIO0 Recovery Entry**
   - Added GPIO0 hold-low boot detection as a recovery-mode entry path.
   - Boot-count recovery remains available as a second recovery path.
12. **Long LED Strip Visibility**
   - Output table now displays LED count plus required universe count, e.g. `500 LEDs / 3U`.
   - This documents the field-installation use case where one controller output drives a long strip across multiple Art-Net/sACN universes.
13. **Per-Output Start Address for Small Devices**
   - Added `OutputChannel::start_address` with JSON load/save and Web UI edit/save support.
   - Shared DMX mapper now treats LED strips as consecutive full-universe consumers, DMX Serial as a full 512-channel universe output, and small outputs as addressable channel ranges inside their `start_universe`.
   - Example: Relay at `U0:CH1`, dimmer at `U0:CH2`, WiZ RGB at `U0:CH10`, and Stepper 16-bit at `U0:CH20` can coexist in the same universe.
   - Mapping also works with ESP-NOW fragmented chunks because overlap is calculated with `dataOffset`.
14. **User Manual Update**
   - Added explicit `start_universe` vs `start_address` explanation.
   - Added byte/channel mapping for WiZ, Relay, Dimmer, PWM Dimmer, DC Motor, Stepper, Servo, and Solenoid.
   - Reframed Standalone/Failsafe/Sequences as a local program engine for controllers running without a live network or after signal loss.
15. **Windows BIN Build Script**
   - Added `build_firmware_bin.bat`.
   - Script locates PlatformIO at `%USERPROFILE%\.platformio\penv\Scripts\pio.exe`, runs `pio run`, creates `dist` if needed, and copies `.pio\build\wt32-eth01\firmware.bin` to a timestamped file.
   - Verified output: `dist\firmware_20260615_212459.bin`.
16. **Output Device Correctness Pass**
   - Fixed `OutputControl::clearChannels()` so it only calls `dmx_driver_delete()` for `type == 1` DMX outputs. Motion outputs reuse `dmxPort` to store LEDC/stepper indexes, so treating those values as DMX ports was unsafe during reload/cleanup.
   - Updated ArtPollReply universe discovery to include every configured output type, not only LED, DMX, and WiZ.
   - Added `wizCtrl.updateDmxData()` after standalone/failsafe processing so WiZ bulbs follow standalone effects and blackout.
   - Added Web UI Start Address overflow validation based on each output's byte count.
17. **OTA Update Fix**
   - Reworked `/update` response logic in `src/main.cpp`: success returns HTTP 200, failure returns HTTP 500 with `Update.errorString()`.
   - Replaced immediate restart inside the request callback with `restartAfterOtaTask()` so the browser can receive the response before reboot.
   - `Update.begin()` now uses `(ESP.getFreeSketchSpace() - 0x1000) & 0xFFFFF000` with `U_FLASH`.
   - Added a second Web UI `flashOta()` definition in ASCII near the end of `web_pages.h` so the active browser function displays server-side OTA errors.
18. **English-only Web UI**
   - Replaced Thai ESP-NOW and OTA text with English.
   - Replaced status symbols with ASCII labels such as `ETH ON` / `ETH OFF`.
   - Removed duplicate OTA JavaScript and non-ASCII separator comments.
19. **Protocol Settings UI**
   - Added a Protocol Settings section to the Network tab.
   - Fields: `artnet_universe`, `sacn_enabled`, and `sacn_multicast`.
   - Updated Web UI `loadSettings()` and `saveSettings()` so these settings persist correctly.
20. **OTA Follow-up Fix**
   - Restored `otaFileSelected()` in `web_pages.h` so selecting a `.bin` enables the Flash Firmware button again.
   - OTA upload now sends `fd.append('update', f, f.name)`.
   - Added `otaInProgress` in `src/main.cpp` to temporarily pause network and output tasks during OTA writes.
21. **Wi-Fi Auto-Reconnect**
   - Added `wifiClientConfigured()` and `reconnectWiFiClient()`.
   - `setupNetwork()` now sets `WiFi.persistent(false)` and `WiFi.setAutoReconnect(true)`.
   - `ARDUINO_EVENT_WIFI_STA_DISCONNECTED` triggers reconnect immediately.
   - `networkTask` retries Wi-Fi reconnect every 10 seconds if STA is still disconnected.
22. **Firmware Version Display**
   - Added `getFirmwareVersion()` in `src/main.cpp`; it formats compile time into `firmware_YYYYMMDD_HHMMSS`.
   - `/api/settings` now includes `firmware_version`, `build_date`, and `build_time`.
   - The Update tab displays the current firmware version.
   - `build_firmware_bin.bat` and `scripts/build_ota.ps1` now run `pio run -t clean` before building so the embedded build time is refreshed.
