# Handover 1.25.00 — 2026-06-20

Reference: [1.24.00](./1.24.00.md) — Session: Code Audit & Bug Fixes

---

## Session Summary
Added separate ESP32 GPIO pin configuration for each segment of the direct-drive 7-segment displays (Type 12 & 13) to avoid requiring contiguous GPIO pins on the WT32-ETH01. Also added two 7-segment PWM dimming modes (Char + Dimming), updated the hardware resource monitor score to display as rounded integers, verified expander support, and registered all modifications to the AI Skill index.

## Key Changes
1. **Resource Score Integer Display:**
   - Modified `updateHwCard()` and the score bar text rendering in `web/index.html` to round resource scores to integers using `Math.round()`.

2. **Separate Pin Configuration for 7-Segment Direct-Drive:**
   - Added `seg_pins[8]` array inside `OutputChannel` struct (`include/output_control.h`) to store separate pin configurations for segments A-G and DP.
   - Updated `loadChannels()` and `saveChannels()` serialization to load and save `seg_pins` array.
   - Modified direct drive GPIO initialization (`motion_control.h`) to configure separate pins from `seg_pins` when defined (not `255`), falling back to contiguous pins.
   - Updated direct drive GPIO write loop in `motion_control.h` to drive separate segment pins.
   - Modified the HTML configuration generator (`web/index.html:renderPinRows()`) to render individual dropdown fields (`no_seg_pin_1` to `7`) for segments B-DP when source is ESP32 GPIO. Segment A maps to the main pin (`no_pin`).

3. **Expander Direct Drive Support:**
   - Re-verified expander (MCP23017, TCA9555, PCF857x) direct drive capability.
   - The UI correctly displays and validates contiguous expander channel options if selected.

4. **PWM Dimming Modes (2 Modes):**
   - Added 2 new display modes (`mc_mode` 4 and 5) in `web/index.html` and `include/motion_control.h`:
     - **Direct 7-pin Dim (2 Ch):** DMX Channel 1 controls segment character, Channel 2 controls PWM duty cycle brightness level.
     - **Direct 8-pin Dim (2 Ch):** DMX Channel 1 controls segment character (including DP), Channel 2 controls PWM duty cycle brightness level.
   - Updated `outputByteCount` and `outputChannelCount` inside `web/index.html` to allocate 2 channels for modes 4 and 5.
   - Updated C++ update loop in `motion_control.h` to read DMX Channel 2 and apply duty cycles dynamically when dimming mode is active.

5. **GPIO Reservation & Duplicate Validation:**
   - Modified `outputGpios(o)` in `web/index.html` to collect and reserve all custom segment pins.
   - Modified `outputJsonUsesPin` and `outputsHaveDuplicateGpio` in `src/main.cpp` to validate segment pins against duplicates and reserved pins (e.g., Ethernet RMII, I2C, Status LED).

6. **Auto-Assignment updates:**
   - Modified `autoAssignOutputPins()` in `web/index.html` to automatically suggest free output GPIOs for segments A-DP when adding a new direct drive display.

7. **AI Skill registration:**
   - Registered all updates to the AI Skill index in `.opencode/skills/esp32-firmware/SKILL.md`.
