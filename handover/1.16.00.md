> [!IMPORTANT]
> **Session: Bug Fixes & Stability Improvements (P0 & P1 Matrix)**
> - **C1 & C2 (AC Dimmer):** Fixed ODR violations by moving static globals in [dimmer_control.h](./include/dimmer_control.h) to externs, defined in [main.cpp](./src/main.cpp). Replaced non-IRAM-safe `digitalWrite()` with `gpio_set_level()`.
> - **C3 & C4 (ESP-NOW):** Eliminated Xtensa alignment exceptions by parsing unaligned data with `memcpy`. Deferred the processing of ESP-NOW packets from the callback ISR context to the main Core 1 `outputTask` using a FreeRTOS Queue (`espNowQueue`), resolving race conditions.
> - **C7 (Division by Zero):** Guarded all calculations involving `sysCfg.output_fps` with a check against 0. Enforced `output_fps > 0` during settings save validation.
> - **C8 (OOM validation):** Added NULL-checks on all `calloc()` calls allocating DMX buffers to prevent dereference-nullptr crashes on OOM.
> - **V1 (HTTP OOM DoS):** Restricted HTTP request bodies inside `collectRequestBody()` to a maximum of 64KB, rejecting larger payloads with HTTP 413.
> - **UART Conflict Resolution:** Added dynamic UART allocation in `setupChannels()`. If DMX is using UART2 (`DMX_NUM_2`), the DFPlayer module is assigned to `Serial1` (UART1). Added API validation to block saving if more than 2 UART channels (DMX on local GPIO + DFPlayer) are configured.
> - **ZC Pin Floating:** Implemented a 5ms Zero-Cross interrupt rate-limiting check inside `onZeroCross()` using `esp_timer_get_time()`.
> - **Scoring Mismatch:** Synchronized Javascript scoring rules with C++ ones in [web_pages.h](./include/web_pages.h) for types 5, 6, and 10.
> - **Compilation:** Build compiles successfully on PlatformIO (`pio run`) with `RAM: 17.5%` and `Flash: 65.2%`.

## 1. ผลลัพธ์การแก้ไขและโครงสร้างที่เปลี่ยนแปลง (Fix Results & Changes)

### 1.1 ตัวแปรส่วนกลางของ Dimmer Control (ODR Violations)
ตัวแปร `dimmerTimer`, `dimmer_tick`, `dimmerChannels`, `numDimmerChannels`, และ `dimmerEnabled` ได้รับการประกาศแบบ `extern` ใน `dimmer_control.h` และย้ายไปนิยามค่าเริ่มต้นไว้ที่ `src/main.cpp` เพื่อป้องกันการเกิดสำเนาตัวแปรซ้ำซ้อนในแต่ละ translation unit

### 1.2 การส่งผ่านข้อมูล ESP-NOW ที่ปลอดภัย (Thread Safety & ISR Deferral)
1. **คอลแบค (`onDataRecv`):** ทำหน้าที่เพียงแค่ก๊อปปี้ข้อมูลแพ็กเก็ตเข้ามายังตัวแปรโครงสร้างแบบ aligned ด้วย `memcpy` จากนั้นส่งเข้าไปยัง FreeRTOS Queue (`espNowQueue`) โดยตรงโดยไม่มีการเรียกใช้งานฟังก์ชันจัดการข้อมูล
2. **การประมวลผลข้อมูล (`outputTask` บน Core 1):** วนรอบอ่านคิว `espNowQueue` เพื่อประมวลผลข้อมูล DMX และสั่งอัปเดตเอาต์พุตอย่างปลอดภัยใน Context ของ Core 1 ป้องกันปัญหา Race Condition และ Cache Exception

### 1.3 การจัดการและจัดสรรพอร์ต UART (DMX + DFPlayer UART Conflict)
ระบบจะตรวจสอบความต้องการใช้งานพอร์ต DMX UART ก่อน จากนั้นจึงเลือกพอร์ตสำหรับ DFPlayer:
- หากใช้ DMX 0 ช่อง: DFPlayer ใช้ `Serial2`
- หากใช้ DMX 1 ช่อง: DMX ได้พอร์ต `DMX_NUM_2` (UART2), DFPlayer ใช้ `Serial1` (UART1)
- หากใช้ DMX 2 ช่อง: DMX ได้พอร์ต `DMX_NUM_2` และ `DMX_NUM_1`, DFPlayer ไม่มีพอร์ตเหลือ (แสดงข้อผิดพลาดและไม่อนุญาตให้เซฟผ่าน API Validation)
