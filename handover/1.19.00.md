## 1. Thread-Safety Flag Modernization (std::atomic)

เพื่อขจัดโอกาสเกิดปัญหา Race Condition (ข้อมูลขัดแย้งกันระหว่างคอร์) ในการควบคุมและประมวลผลข้อมูลเฟรม DMX ระหว่าง Core 0 (รับสัญญาณเครือข่าย/ESP-NOW Callback) และ Core 1 (งานประมวลผลเอาต์พุต `outputTask`):
1. นำเข้า `<atomic>` ในไฟล์:
   - [output_control.h](./include/output_control.h)
   - [artnet_control.h](./include/artnet_control.h)
   - [espnow_control.h](./include/espnow_control.h)
   - [main.cpp](./src/main.cpp)
2. อัปเกรดตัวแปรส่งสัญญาณข้ามคอร์เป็น `std::atomic<bool>`:
   - `networkFramePending` (ประกาศใน `output_control.h`, กำหนดค่าใน `main.cpp` เป็น `std::atomic<bool> networkFramePending(false)`)
   - `ArtNetControl::newRxData` (เปลี่ยนเป็น `std::atomic<bool>`)
   - `EspNowControl::newRxData` (เปลี่ยนเป็น `std::atomic<bool>`)

## 2. การปรับปรุงระบบ Reboot และ Diagnostics บน Web UI ([web_pages.h](./include/web_pages.h))

1. **Reboot Overlay Countdown:** เพิ่ม UI สำหรับนับถอยหลังเมื่อเกิดการรีบูตบอร์ด (เช่น บันทึกการตั้งค่า, รีเซ็ตช่องสัญญาณเอาต์พุต, นำเข้าไฟล์ตั้งค่า หรือทำ OTA) เพื่อให้ผู้ใช้งานทราบสถานะของระบบอย่างชัดเจนและหน้าเว็บจะทำการ Reload ใหม่เมื่อครบกำหนดเวลาโดยอัตโนมัติ
2. **Diagnostics Timer:** ปรับเปลี่ยนให้หน้า Diagnostics ดึงข้อมูลสถานะแบบเรียลไทม์ทุก ๆ 5 วินาทีแทนการกด Refresh แมนนวล ช่วยอำนวยความสะดวกในการเฝ้าระวังประสิทธิภาพการทำงานของบอร์ด
