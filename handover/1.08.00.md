## Changes Made in v1.8.0 (Ongoing)
1. **sACN E1.31 Integration**:
   - Implemented `SACNControl` handling UDP port 5568 unicast & multicast streams.
   - Built ACN Root, Framing, and DMP layer validations.
   - Added source priority tracking (keeps data only from the highest priority source) and source timeout/expiration (2.5s).
   - Exposed status endpoints under `/api/sacn/status` and live status telemetry.
2. **Defined Control Instances in main.cpp**:
   - `MotionControl motionCtrl;`
   - `DimmerControl dimmerCtrl;`
3. **Verified Hack Removal**:
   - Confirmed `ch.ledc_chan2` is used everywhere in `motion_control.h` instead of reusing `ch.wiz_map_mode` for DC motor PWM + PWM.
4. **Motion UI Contextual Settings in web_pages.h**:
   - Added wrapper IDs to specific input elements.
   - Refactored `toggleOutFields()` to selectively display settings (resolution, sub-modes, speed/frequency, deadbands, limits, steps, direction invert, and brake settings) depending on motor type.
   - Customized field labels dynamically (Frequency vs Speed).
   - Displayed `mc_brake` checkbox only for DC Motor sub-modes supporting it (PWM+PWM and IN1+IN2+EN).
5. **Stepper Auto-Zero Homing Support**:
   - Expanded `OutputChannel` in `output_control.h` with pin4, homing direction, mode, speed, and timeout parameters.
   - Developed homing command state machine inside `motion_control.h` driven by Ch N+2/N+3 (depending on 8/16-bit position resolution).
   - Implemented Stop (100-199) and Homing (200-255) command debounce (>1s).
   - Supported Sensor Homing (Mode A) searching in configured direction until active-low pin 4 triggers.
   - Supported Time Homing (Mode B) running in homing direction until timeout expires.
   - Successfully called `stepper->forceStopAndNewPosition(0)` to zero target position.
   - Added configuration fields to Web UI with dynamic visibility.
6. **Failsafe & Standalone Unification**:
   - Renamed "Standalone" tab button to "Failsafe & Automation" and removed the "Sequences" tab button from `web_pages.h`.
   - Relocated Custom JSON Sequence steps editor HTML to a nested container (`#sa-seq-container`) within the failsafe/standalone panel.
   - Updated JavaScript `toggleSaColor()` to dynamically show the sequence editor container only when `sa_effect` is set to `4` (Custom JSON Sequence) and hide it for other modes.
7. **Enhanced LED & DMX Effects**:
   - Integrated Moving Gradient (Effect 5), Color Chase (Effect 6), and DMX Slow Ramp (Effect 7) logic inside `standalone_control.h`.
   - Built math-driven rendering logic using sine wave interpolation for gradients, trailing tail decay for chases, and phase-shifted triangle waves for DMX channel ramps.
   - Updated Web UI dropdown and JavaScript `toggleSaColor()` to contextually display the solid color picker for Solid Color, Moving Gradient, and Color Chase effects.
8. **WiFi Client Static IP Integration**:
   - Added config fields `wifi_dhcp`, `wifi_ip`, `wifi_netmask`, `wifi_gateway`, `wifi_dns` in `config.h`.
   - Setup configuration loading and saving from Preferences NVS.
   - Initialized static Wi-Fi configuration via `WiFi.config` in `setupNetwork()` inside `main.cpp`.
   - Integrated Wi-Fi settings in the GET/POST `/api/settings` handlers.
   - Implemented static Wi-Fi setup fields and DHCP toggle logic (`toggleWifiIp`) in `web_pages.h`.
9. **NTP/Schedule Tab Separation in web_pages.h**:
   - Added new Web UI tab button `Schedule & NTP` and corresponding HTML `pane-sched` containing form inputs.
   - Moved all **NTP Time Sync** and **Schedule** setup elements from `pane-sa` (Standalone) into the new `pane-sched` form container.
   - Fixed checkbox state collection in JavaScript `saveSettings(e)` so unchecked settings outside the active form are not overwritten as `false` on submission.
