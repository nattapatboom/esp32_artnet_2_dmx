# Project Handover v1.5.0 (End of Phase 5)

## Current Status
- **Phase 5 Completed**: We have fully integrated Relay, AC Dimmer, and RMT-based Extra DMX functionality. 
- **Project Structure**:
  - `dimmer_control.h` (NEW): Introduces hardware timer interrupts (Timer 0) and Zero-Crossing (ZC) external interrupts. Triggered every 39 microseconds to deliver precision phase-angle triac firing for up to 8 Dimmer channels.
  - `rmt_dmx.h` (NEW): Extends the DMX hardware capability beyond the 2 native UARTs. Implements bit-banging of DMX-512 standard over the RMT peripheral, including 100us Break and 12us MAB.
  - `output_control.h`: 
    - `updateRelays()` added to evaluate DMX>127 threshold for `CHAN_TYPE_RELAY` (type 2).
    - Framerate throttling implemented natively using `output_fps`.
    - Handles automatic fallback to `RmtDmxDriver` when more than 2 DMX outputs are requested.
  - `web_pages.h`:
    - Updated UI `select` drivers: Added Relay, AC Dimmer.
    - Updated Global Output FPS (formerly DMX FPS).
    - Added Global Zero-Crossing GPIO mapping.
  - `test/test_rmt_dmx/`: Dedicated Arduino sketch strictly for testing RMT DMX signals via an oscilloscope if needed.
- **System Global State**: Stable, flash usage has increased slightly due to the RMT DMX inclusion but still well within bounds.

## Known Limitations / Reminders
- Dimmers run purely on Hardware Timer 0 Interrupts inside IRAM. Be careful of anything else trying to bind to Timer 0, Group 0.
- 50Hz mains is assumed for Dimmer delay calculations (`DIMMER_TICK_US 39`). If 60Hz is used, `DIMMER_TICK_US` must drop to ~33.
- RMT channels are maxed at 8. If 8 LED strips are configured, DMX channel 3 will fail to grab an RMT.
