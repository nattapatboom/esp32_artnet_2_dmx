# Project Handover v1.6.0 (End of Phase 5.5)

## Current Status
- **Phase 5.5 Completed**: The ESP32 Art-Net node is now a fully capable Motion Controller. It can natively drive various actuators directly from DMX.
- **Project Structure**:
  - `motion_control.h` (NEW): Manages hardware assignments for PWM Dimmers, DC Motors (H-Bridge), RC Servos, and Stepper Motors. It dynamically allocates ESP32's native LEDC channels (up to 16) and interfaces with the `FastAccelStepper` library.
  - `output_control.h`: Extended `OutputChannel` struct with 11 new parameters (`mc_resolution`, `mc_mode`, etc.) and handles saving/loading from `/outputs.json`.
  - `web_pages.h`: Advanced UI updates to present a dynamic "Motion Control Parameters" card when specific motor types are selected. Hides irrelevant pins.
  - `platformio.ini`: Added dependency `gin66/FastAccelStepper @ ^0.30.13`.

## Known Limitations / Reminders
- DMX Multi-byte resolution (e.g., 16-bit) reads data as Big Endian (MSB first) from the Art-Net stream.
- `FastAccelStepper` might conflict with ESP32's RMT if 8 LED Strips are already configured. Ensure users know the hardware limits.
