> [!IMPORTANT]
> **Session: I2C Lock Protection, GPIO 12 Boot Warning, and Contiguous Output Types Layout Versioning**
> - **I2C Hang Protection:** Replaced indefinite blocks (`portMAX_DELAY`) on `i2cMutex` takes with a 100ms timeout check (`xSemaphoreTake(i2cMutex, pdMS_TO_TICKS(100)) == pdTRUE`) across I2C scan, PCA9685, digital expanders, and display drivers. Prevents board lockups and watchdog resets if the I2C SCL/SDA lines hang due to hardware issues.
> - **Bootstrapping Pin Warning (GPIO 12):** Integrated active warning boxes in the Web UI that trigger when the MTDI bootstrapping pin (GPIO 12) is configured for status LED, ZC pin, I2C bus, or any output channel, helping users avoid potential boot loops.
> - **Contiguous Output Types Migration & Versioning:** Renumbered and re-indexed all C++ and JavaScript output type checks to be completely contiguous from 0 to 13. To prevent configuration corruption due to double-migration on reboot, introduced a layout version field (`version: 2`) in `outputs.json`. The firmware automatically migrates older configurations upon startup and writes back the versioned file.
> - **Compilation:** Verified successful firmware compilation using PlatformIO.

## 1. I2C Hang Protection (100ms Lock Timeout)

เพื่อป้องกันปัญหาบอร์ดล็อกชะงัก (Deadlock) หรือเกิดการรีสตาร์ตจาก Watchdog เมื่อบัส I2C เกิดปัญหาการติดต่อทางกายภาพ (เช่น สายหลุด, สัญญาณรบกวน, บัสค้าง):
1. อัปเกรดการดึงสิทธิ์ใช้บัสผ่าน `i2cMutex` ในไฟล์:
   - [pca9685_control.h](../include/pca9685_control.h)
   - [i2c_gpio_expander.h](../include/i2c_gpio_expander.h)
   - [display_driver.h](../include/display_driver.h)
   - [main.cpp](../src/main.cpp) (ในส่วนงานสแกนอุปกรณ์ I2C)
2. เปลี่ยนจากการเรียกแบบปิดกั้นถาวร `xSemaphoreTake(i2cMutex, portMAX_DELAY)` ไปเป็น `xSemaphoreTake(i2cMutex, pdMS_TO_TICKS(100)) == pdTRUE`
3. คืนค่าหรือข้ามขั้นตอนคำสั่งส่งข้อมูลทันทีเมื่อดึงสิทธิ์ไม่สำเร็จภายใน 100ms เพื่อรักษาความราบรื่นในการทำงานของระบบหลัก

## 2. ระบบป้องกัน Bootloop จากขา Bootstrapping GPIO 12

1. เพิ่มระบบแจ้งเตือนเชิงรุกบนหน้าเว็บเอาต์พุตและระบบตรวจสอบความขัดแย้งของขาขา GPIO หากผู้ใช้งานนำเอา GPIO 12 (MTDI) ไปตั้งค่าในส่วนต่าง ๆ:
   - เอาต์พุตช่องสัญญาณ (เอาต์พุตเดี่ยว, เอาต์พุตแบบกลุ่ม/มอเตอร์/RGB, 7-Segment, ฯลฯ)
   - Status LED GPIO
   - Zero-Crossing GPIO
   - I2C SDA / SCL GPIO
2. ขา GPIO 12 เป็นขา Bootstrapping ทำหน้าที่ระบุระดับแรงดันไฟฟ้าสำหรับแฟลชชิป (VDD_SDIO 1.8V vs 3.3V) หากขานี้ถูกดึงสูง (HIGH) ในระหว่างเริ่มระบบ บอร์ดจะล้มเหลวในการเริ่มทำงานหรือติดลูปการบูตทันที

## 3. การจัดเรียงหมายเลขประเภทเอาต์พุต (Contiguous Output Types 0-13)

เพื่อลบช่องว่างจากการถอดเอาต์พุตประเภทเก่า (WiZ และ Virtual 7-Segment) และรองรับระบบในระยะยาว จึงได้ลดประเภทจาก 0-15 ให้เหลือ 0-13 แบบเรียงต่อเนื่อง:
- `0`: LED Strip
- `1`: DMX Output
- `2`: Relay
- `3`: AC Dimmer
- `4`: DC PWM Dimmer
- `5`: DC Motor
- `6`: Stepper
- `7`: RC Servo
- `8`: Solenoid
- `9`: Analog RGB
- `10`: Buzzer
- `11`: Smoke Shooter
- `12`: 7-Segment
- `13`: DFPlayer MP3

### Layout Versioning & Safety Migration (C++ / JS)
เพื่อป้องกันปัญหาการแปลข้อมูลซ้ำซ้อน (Double-Migration) บนตัวแปรประเภทเอาต์พุตแบบลำดับเก่าในตอนโหลดหน้าเว็บหรือรีบูต:
1. กำหนดโครงสร้างเวอร์ชันของคอนฟิกูเรชัน `"version": 2` ลงในรูทของไฟล์ข้อมูล `outputs.json`
2. **C++ (output_control.h):** ตรวจสอบเวอร์ชันของเอาต์พุตผ่าน `"version"` หากต่ำกว่า 2 หรือไม่มีเวอร์ชัน จะดึงค่าขึ้นมาแปลงตามตารางคู่เทียบประเภทเก่าไปยังใหม่ แล้วสั่งบันทึกอัปเดตลงบอร์ดพร้อมระบุเวอร์ชัน 2 ทันที
3. **C++ (main.cpp):** อัปเดตเงื่อนไขตรวจสอบขาซ้ำ, ตรวจสอบประเภทที่ยอมรับได้, และตัวช่วยตรวจสอบขัดแย้งขาเอาต์พุต/อินพุต ให้สอดคล้องกับตารางประเภท 0-13 ตัวใหม่ทั้งหมด
4. **JS (web/index.html):** ปรับเงื่อนไขการนับแชนแนลทรัพยากร, การเปิด/ปิดหน้าต่างตั้งค่าขา, การประเมินคะแนนทรัพยากร, และการนับไบต์ ให้เรียงตาม 0-13 และทำการส่งข้อมูลในรูปแบบเวอร์ชัน 2 เสมอเมื่อบันทึกคอนฟิก
