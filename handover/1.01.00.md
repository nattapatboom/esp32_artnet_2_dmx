# Handover เอกสารส่งมอบงาน v1.1.0: Art-Net Node Device (WT32-ETH01)

เอกสารฉบับนี้สรุปความคืบหน้าและการพัฒนาในเวอร์ชัน **v1.1.0** ซึ่งเน้นการปรับปรุงความยืดหยุ่นในการตั้งค่าช่องสัญญาณเอาต์พุต DMX ให้ทำงานแบบไดนามิก ปรับปรุงหน้าเว็บให้อ่านง่ายขึ้น และระบบการจัดการหลอดไฟ WiZ และการค้นหาสัญญาณ Wi-Fi ลูกข่าย

---

## 1. ฟีเจอร์ที่เสร็จสมบูรณ์ในเวอร์ชัน v1.1.0 (Completed Features)

* **ระบบค้นหา Wi-Fi ลูกข่าย (Asynchronous Wi-Fi Scanner)**:
  * พัฒนา REST API `/api/wifi-scan` แบบ Asynchronous เพื่อค้นหาสัญญาณ Wi-Fi ในบริเวณรอบๆ
  * เพิ่มปุ่ม **Scan** ในหน้าการตั้งค่าเครือข่าย เมื่อกดจะสแกนหาเครือข่ายไร้สาย และแสดงเป็นตัวเลือกให้คลิกเลือกชื่อ SSID ได้ทันทีโดยไม่ต้องพิมพ์เอง

* **การจัดการ DMX เอาต์พุตแบบไดนามิก (Dynamic DMX Output Channels)**:
  * จากเดิมที่ DMX ถูกระบุแบบ Hardcoded ในตัวเครื่องผ่านพิน GPIO 17 และจักรวาลที่ 0 เท่านั้น ในเวอร์ชันนี้ได้รับการปรับปรุงใหม่ให้ DMX มีสถานะเป็น **ช่องเอาต์พุต (Output Channel Line)** ในตารางเอาต์พุต เช่นเดียวกับไฟเส้น LED
  * สามารถเพิ่มและลดสายเอาต์พุต DMX ได้ตามต้องการในตารางการตั้งค่าหน้าเว็บ
  * รองรับการจ่ายเอาต์พุต DMX พร้อมกันสูงสุด 2 ช่อง โดยระบบจะสลับใช้โมดูล Hardware UART อัตโนมัติ (ช่องแรกใช้ UART2/`DMX_NUM_2` และช่องสองใช้ UART1/`DMX_NUM_1`)
  * แต่ละสาย DMX สามารถกำหนดขาสัญญาณเอาต์พุต (Pin) และจักรวาลเริ่มต้น (Start Universe) แยกกันเป็นอิสระ

* **การจัดการหลอดไฟ WiZ แบบไดนามิก (Dynamic WiZ Bulbs List)**:
  * ย้ายระบบตั้งค่าหลอดไฟ WiZ จากการบันทึก NVS แบบ Hardcoded ค่าเดียว มาเป็นการจัดการผ่านตารางบนหน้าเว็บ
  * บันทึกข้อมูลลงแฟลชการ์ดด้วย `/wiz_bulbs.json` บน LittleFS
  * สามารถเพิ่ม ลบ และกำหนดไอพี, พอร์ต UDP, DMX แชนเนลเริ่มต้น และโหมดการแปลงค่าของหลอดไฟแต่ละดวงแยกกันได้ทันที โดยไม่มีการรีบูตบอร์ด (Instant Apply)

* **ปรับปรุงการอ่านข้อความเมนูแบบเลือก (Dropdown Visibility Fix)**:
  * แก้ไขปัญหาของบราวเซอร์บนมือถือหรือระบบปฏิบัติการบางรุ่นที่แสดงผลตัวเลือกเมนูแบบเลือก (Dropdown Option) อ่านไม่รู้เรื่อง (เช่น ตัวอักษรสีขาวบนพื้นหลังสีขาว หรือข้อความมืดมองไม่เห็น)
  * เพิ่มโค้ด CSS ตกแต่ง `select option` ให้ใช้สีพื้นหลังเข้มและตัวอักษรสีขาว ทำให้อ่านง่ายทุกอุปกรณ์

## 4. สิ่งที่ต้องทำต่อในเวอร์ชันถัดไป (Next Steps)
1. **การติดตั้งใช้งานร่วมกับบอร์ดแปลงสัญญาณจริง**: ทดสอบต่อสาย DMX เอาต์พุตออกจากขา GPIO ต่างๆ (เช่น GPIO 17, GPIO 12) ผ่านวงจรแปลงระดับสัญญาณ RS485 แล้วนำไปต่อควบคุมโคมไฟเคลื่อนที่ (Moving Head) หรือโคมไฟพาร์ (DMX LED Par)
