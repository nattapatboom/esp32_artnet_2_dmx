## 1. Default Channel Type 0 → 3

ก่อนหน้านี้ทุกจุดที่ใช้ default channel จะเป็น `type = 0` (AC Dimmer) แทนที่จะเป็น `type = 3` (RGB LED) ซึ่งเป็น default ที่ตั้งใจไว้

แก้ที่:
- `include/output_control.h` — `loadChannels()` → OOM fallback + file-not-found path (2 ที่)
- `src/main.cpp` — GET `/api/outputs` (no file fallback), `/api/outputs/clear`, `/api/config/factory-reset`, `/api/config/backup` (fallback empty)

## 2. Solenoid Save Dead Code (output_control.h)

```cpp
// ก่อนหน้า (BROKEN):
if ((ch.type >= 4 && ch.type <= 8) || ch.type == 17) {  // type 17 อยู่ที่นี่
    ...
} else if (ch.type == 17) {  // ← ไม่มีทางถูกรัน! dead code
    item["solenoid_mode"] = ...
}

// หลังแก้ (FIXED):
if (ch.type >= 4 && ch.type <= 8) {  // ลบ type 17 ออกจาก range
    ...
} else if (ch.type == 17) {  // ← ถูกรันได้แล้ว
    item["solenoid_mode"] = ...
}
```

ผลกระทบ: ก่อนหน้า solenoid_mode, threshold, pulse_ms, pre_delay, post_delay ไม่ถูก save เลย → หลัง reload ค่าทุกอย่างหาย

## 3. Buzzer mc_freq ไม่ถูก Save

Buzzer (type 9) ใช้ LEDC และ `mc_freq` แต่ไม่มี save block → หลัง reload `mc_freq = 1000` (default) เสมอ

```cpp
// เพิ่ม:
} else if (ch.type == 9) {
    item["mc_freq"] = ch.mc_freq;
}
```

## 4. Motor Mode Redundant Check (motion_control.h)

else if (ch.mc_mode == 1) { ... }

## 5. DMX Channel Score ผิดใน Web UI (web/index.html)

`channelScore()` function สำหรับ DMX (type 1) นับ `uart=1` เสมอ แม้ว่า UART อาจเต็มแล้วทำให้ DMX ตกไปใช้ RMT (ที่ถูกกว่า)

```js
// ก่อนหน้า: uart=1 เสมอ (ผิดเมื่อ UART เต็ม)
case 1: if(src===0) gpio=1; uart=1; break;

// หลังแก้: estimate ตาม DFPlayer count
const dfCount = outputs.filter(x => typeId(x) === 10).length;
const freeUarts = Math.max(0, 2 - dfCount);
if (dmxBefore < freeUarts) uart = 1; else rmt = 1;
```
