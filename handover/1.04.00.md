# Project Handover v1.4.0 (End of Phase 4)

## Current Status
- **Phase 4 Completed**: We have implemented the Fail-safe Recovery Mode and integrated WiZ Smart Bulbs into the primary `OutputChannel` structure.
- **Project Structure**:
  - `main.cpp`: Manages `RTC_DATA_ATTR bootCount`. Triggers Recovery Mode AP and OTA if reset 3 times in 3 seconds. Removed legacy WiZ JSON endpoints.
  - `output_control.h`: Added `wiz_ip`, `wiz_map_mode` and state variables for `CHAN_TYPE_WIZ`.
  - `wiz_control.h`: Refactored to listen to `outputCtrl.getChannels()` rather than maintaining a separate JSON.
  - `web_pages.h`: Integrated WiZ Bulbs into the main Output Driver table. Removed the old separate WiZ tab. Removed global `wiz_enabled` toggle checkbox (WiZ UDP bridge now automatically opens if any type 4 channel exists).
  - `config.h`: Legacy `sysCfg.wiz_enabled` is no longer actively checked in the main loop to decide if the UDP bridge should open.
- **Build**: Successfully compiles with PlatformIO without any errors. RAM usage is stable.

## Next Steps (Phase 5)
- **Phase 5: Relay, AC Dimmer & Extra DMX (Hardware Intensive)**
  - `CHAN_TYPE_RELAY=2`
  - `CHAN_TYPE_DIMMER=3` using Zero-Crossing Interrupts + HW Timer.
  - **RMT DMX**: Move extra DMX channels off of `SoftwareSerial` to ESP32's `RMT` peripheral for precision and performance without FreeRTOS jitter.

## Known Limitations / Reminders
- WiZ implementation broadcasts to the designated IP Address directly at 10Hz.
- `led_count` limit warning in UI is pending for Phase 6.
