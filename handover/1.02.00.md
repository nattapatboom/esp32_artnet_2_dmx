## 1. การเพิ่มประสิทธิภาพที่ทำในเวอร์ชัน v1.2.0 (Performance Optimizations)

### 1.1 `output_control.h` — LED Render Loop Optimization
- **ปัญหาเดิม**: ลูปการอัปเดต LED (`updateLeds`) ทำการคำนวณ `i / 170` และ `i % 170` (Integer Division + Modulo) ซ้ำทุก Pixel ทุก Frame ซึ่งเป็น Operation ที่ช้า
- **การแก้ไข**: เปลี่ยนเป็นใช้ Counter สะสม (`posInUniverse`, `bufOffset`) แทน เพื่อเลื่อน Universe Boundary โดยไม่ต้องหารซ้ำ ลด CPU Load ในลูปหลักได้อย่างมีนัยสำคัญสำหรับ LED Count สูง
- **การแก้ไขเพิ่มเติม**: Cache ค่า `sysCfg.device_mode`, `sysCfg.sa_brightness`, และ `ch.color_order` ไว้ใน Local Variable ก่อนเข้าลูป เพื่อหลีกเลี่ยงการอ่าน Global Struct ซ้ำทุก Pixel
- **Timer Fix**: แก้ไข `loop()` ให้จับ `millis()` ไว้ใน Variable เดียว (`now`) ก่อนการเปรียบเทียบ แทนที่จะเรียก `millis()` สองครั้ง (ก่อนและหลัง `sendDmxFrames`) ซึ่งทำให้ Timer เลื่อนทุกรอบ

### 1.2 `standalone_control.h` — Integer HSV + Solid Color Guard
- **ปัญหาเดิม**: ฟังก์ชัน `hueToRGB()` ใช้ Floating-Point Arithmetic (float/floor) ซึ่งช้ากว่า Integer บน ESP32 แม้จะมี FPU
- **การแก้ไข**: เขียนใหม่เป็น Integer-Only HSV→RGB ใช้ Integer Division ล้วน ผลลัพธ์เทียบเท่ากันสำหรับ 8-bit Color แต่ไม่ต้องโหลด `libm` เพิ่ม
- **ปัญหาเดิม**: Effect Mode 3 (Solid Fixed Color) เรียก `renderSolidColor()` ทุก Tick แม้สีจะไม่เปลี่ยน ทำให้ Fill Buffer 512 Bytes ซ้ำโดยไม่จำเป็น
- **การแก้ไข**: เพิ่ม Dirty-Flag (`lastR/G/B`) ตรวจสอบก่อนว่าสีเปลี่ยนหรือเปล่า ถ้าไม่เปลี่ยนก็ข้ามการ Fill

### 1.3 `wiz_control.h` — Bug Fix: uint8_t Overflow
- **ปัญหาเดิม (Silent Bug)**: `WizBulb.lastSentR/G/B/Dim` เป็น `uint8_t` แต่ Initialize ด้วยค่า `999` ซึ่งเกิน Range และ Overflow ไปเป็น `231` (999 & 0xFF) ทำให้การตรวจ State Change ครั้งแรกอาจผิดพลาดได้ในบางกรณี
- **การแก้ไข**: เพิ่ม Field `bool firstSend = true` ใน Struct แทน แล้วใช้ Flag นี้เพื่อบังคับส่งครั้งแรกโดยไม่ต้องพึ่ง Magic Number ที่ Overflow

### 1.4 `main.cpp` — ArduinoJson v7 API + ESPAsyncWebServer API
- **ปัญหาเดิม**: ใช้ `doc.containsKey()` ซึ่งเป็น API เก่าของ ArduinoJson v6 และถูก Mark Deprecated ใน v7 แล้ว
- **การแก้ไข**: เปลี่ยนทุก `doc.containsKey("key")` เป็น `doc["key"].is<Type>()` ซึ่งเป็น Pattern ที่ถูกต้องสำหรับ v7 และมีความปลอดภัยด้านชนิดข้อมูลสูงกว่า
- **ปัญหาเดิม**: ใช้ `request->send_P()` ที่ถูก Mark Deprecated ใน ESPAsyncWebServer เวอร์ชันใหม่
- **การแก้ไข**: เปลี่ยนเป็น `request->send()` ที่รองรับ PROGMEM String โดยตรงในเวอร์ชันใหม่
