================================================================================
          สรุปแผนงานโครงการ (Project Plan Summary): Art-Net Node Device
================================================================================

1. วัตถุประสงค์และขอบเขต (Objective & Scope)
--------------------------------------------------------------------------------
* พัฒนาอุปกรณ์แปลงสัญญาณ Art-Net (DMX over IP) เป็น DMX512 และ Pixel LED 
* ใช้บอร์ด WT32-ETH01 (ESP32-LAN8720) ผ่านสาย LAN เพื่อความเสถียรสูงสุด
* ออกแบบเพื่อเน้นใช้งานร่วมกับโปรแกรม TouchDesigner (ผ่าน DMX Out CHOP) และ Resolume Arena
* รองรับการติดตั้งในร้านอาหาร, บาร์, คอนเสิร์ต และงานอินเตอร์แอคทีฟ (Interactive Installation)

2. สรุปฮาร์ดแวร์และการเชื่อมต่อ (Hardware Layout & Pinout)
--------------------------------------------------------------------------------
* บอร์ดควบคุมหลัก: WT32-ETH01 (ESP32 Dual-Core 240MHz)
* พอร์ตเอาต์พุตหลักบนบอร์ด: GPIO 4, 12, 14, 15 (ต่อผ่านไอซีบัฟเฟอร์ SN74HCT125N เพื่อแปลงระดับแรงดันสัญญาณจาก 3.3V เป็น 5V และป้องกันตัวบอร์ดควบคุม)
* พอร์ตควบคุมไฟ (DMX512): ต่อชิป MAX485 หลังไอซีบัฟเฟอร์ โดยใช้เฉพาะขา TX ส่งออกอย่างเดียว (TX-Only) และต่อขา DE/RE ของชิปเข้ากับ VCC 5V เสมอเพื่อประหยัดพิน
* ไฟแสดงสถานะบนบอร์ด (Onboard LEDs Hack): 
  - LED3 (GPIO 5 / RXD2): ใช้ซอฟต์แวร์สั่งควบคุมให้กะพริบแสดงสถานะระบบ เช่น โหมดตั้งค่า SoftAP
  - LED4 (GPIO 17 / TXD2): ใช้เป็นขาออก DMX ทำให้ไฟกะพริบถี่ๆ ตามจังหวะข้อมูลวิ่งอัตโนมัติ
* ข้อควรระวังเพิ่มเติม: 
  - หลีกเลี่ยงใช้งานพิน GPIO 16 (ใช้ควบคุมสาย LAN ในบอร์ด)
  - พิน GPIO 12 เป็นพินตรวจสอบสถานะตอนบูต (Bootstrapping) แต่การต่อผ่าน SN74HCT125N ช่วยแก้ปัญหาบูตไม่ขึ้นได้เนื่องจากเป็นพอร์ตความต้านทานสูง (High Impedance)
  - ควรต่อฟิวส์สายไฟบวก และทำการจ่ายไฟเลี้ยงเพิ่ม (Power Injection) ทุกๆ 150-200 ดวงของไฟเส้นเพื่อความปลอดภัยและป้องกันไฟกระตุก/สีเพี้ยน

3. คุณสมบัติเด่นของซอฟต์แวร์ (Core Software Features)
--------------------------------------------------------------------------------
* การเข้าถึงตั้งค่าอัจฉริยะ (Network & Configuration Access):
  - โหมด Ethernet: เชื่อมต่อด้วย DHCP หรือ Static IP
  - โหมดตั้งค่าไร้สาย (Wi-Fi SoftAP): หากไม่พบสาย LAN ตอนเปิดเครื่อง บอร์ดจะเปิดปล่อย Wi-Fi คอนฟิกในตัวอัตโนมัติ (เข้าเว็บที่ IP 192.168.4.1 หรือ http://artnet.local ผ่าน mDNS)
  - ระบบแยกโหมดอัจฉริยะ: หากตั้งบอร์ดเป็นตัวรับสัญญาณไร้สาย (ESP-NOW Slave) บอร์ดจะปิดสัญญาณ Wi-Fi SoftAP เองภายใน 2 นาทีหลังบูต เพื่อป้องกันสัญญาณรบกวนในงานจริง
* ระบบรับส่งสัญญาณ (Art-Net & Wireless Bridge):
  - รองรับ ArtPoll และ ArtPollReply ค้นหาตัวตนเจออัตโนมัติบนเครือข่าย
  - รองรับการเชื่อมต่อข้ามบอร์ดไร้สาย (ESP-NOW Bridge): บอร์ดตัวส่ง (Master) ต่อแลนกระจายสัญญาณต่อให้บอร์ดตัวรับ (Slave) หลายๆ ตัวผ่านวิทยุไร้สายโดยไม่ต้องผ่านเราเตอร์
* โหมดเล่นอิสระและการควบคุมหลอดไฟบ้าน (Standalone & Smart Bulb):
  - โหมด Standalone: ทำงานเมื่อเครือข่ายหลุด (Fail-safe) โดยสามารถเลือกให้ค้างไฟล่าสุด, ดับไฟทั้งหมด หรือเล่นโปรแกรมไฟอัตโนมัติในตัว (บันทึกสเต็ปไฟเป็น JSON ลง Flash Memory ของชิป)
  - ตั้งตรรกะเวลาอัตโนมัติ: ซิงค์เวลาจริงผ่านเน็ตเวิร์ก (NTP) เพื่อตั้งเวลาเปิด-ปิดไฟตามช่วงเวลาที่กำหนดได้ (เช่น 18:00 - 02:00 น.)
  - ควบคุมหลอดไฟ WiZ: แปลงสัญญาณ DMX ส่งคำสั่งผ่านโปรโตคอล WiZ Local UDP (พอร์ต 38899) โดยมีระบบจำกัดความเร็ว 5-10Hz เพื่อไม่ให้หลอดไฟค้าง และรองรับโหมด 3 ช่อง (RGB Only - แปลงระดับหรี่แสงอัตโนมัติ) หรือ 4 ช่อง (RGB+Dimmer)
  - โลโก้ประหยัดความจำขั้นสูง: แปลงโลโก้ `[CHAL]_128_64.bmp` เป็น 1-bit PNG ขนาด 266 ไบต์ แล้วฝังใน HTML ในแบบ Base64 (Inline Data URI 356 ตัวอักษร) ประหยัดแรม (RAM) เป็น 0 ไบต์ และหน้าเว็บโหลดเร็วกว่าเพราะไม่ต้องดาวน์โหลดไฟล์รูปแยก

4. ขั้นตอนการดำเนินงานย่อ (Development Phases Summary)
--------------------------------------------------------------------------------
* ระยะที่ 1 (เตรียมฮาร์ดแวร์): วาดวงจรและต่อบอร์ดทดลองบน Breadboard ตรวจสอบความถูกต้องของ SN74HCT125N และบอร์ดขับเอาต์พุต
* ระยะที่ 2 (สร้างโปรเจกต์): ตั้งค่าระบบ PlatformIO และทดสอบเชื่อมต่อ Ethernet ให้รับ IP
* ระยะที่ 3 (ระบบสื่อสาร): พัฒนาการรับแพ็กเก็ต Art-Net, การรับส่งข้อมูลผ่าน ESP-NOW, และตรรกะสลับเปิด SoftAP เมื่อไม่มีสาย LAN
* ระยะที่ 4 (ไดรฟ์เวอร์ไฟ): พัฒนาซอฟต์แวร์ DMX TX-Only, การขับ Pixel LED ข้ามหลายจักรวาล, ตัวเล่นเอฟเฟกต์ Standalone JSON และระบบส่งคำสั่ง WiZ UDP (จำกัดความเร็วส่ง)
* ระยะที่ 5 (หน้าเว็บตั้งค่า): พัฒนาหน้าเว็บแบบ Responsive (สำหรับ PC/มือถือ) ตั้งค่า IP, เลขพินเอาต์พุตไดนามิก, ตั้งค่า IP หลอดไฟ WiZ, การแมปช่อง และตั้งเวลาทำงานอัตโนมัติ
* ระยะที่ 6 (ทดสอบจริง): ทดสอบการทำงานต่อเนื่องยาวนาน ทดสอบร่วมกับ TouchDesigner (DMX Out CHOP) และจัดทำไฟล์ส่งมอบงาน (Handover) แต่ละเวอร์ชัน
