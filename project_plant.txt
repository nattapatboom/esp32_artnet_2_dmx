================================================================================
          PROJECT PLAN: ESP32 WT32-ETH01 Art-Net to DMX/Pixel LED Device
================================================================================

1. วัตถุประสงค์โครงการ (Project Objective)
--------------------------------------------------------------------------------
พัฒนาอุปกรณ์แปลงสัญญาณ Art-Net (DMX over IP) เป็นสัญญาณ DMX512 หรือควบคุมไฟเส้น Pixel LED 
โดยใช้บอร์ด WT32-ETH01 (ESP32) ผ่านการเชื่อมต่อสาย LAN (Ethernet) เพื่อความเสถียร รวดเร็ว 
และลดสัญญาณรบกวนเมื่อเทียบกับการใช้ Wi-Fi เหมาะสำหรับงานติดตั้งระบบแสงในร้านอาหาร, บาร์, 
คอนเสิร์ต, งานแนวสร้างสรรค์และอินเตอร์แอคทีฟ (Interactive Installation) ด้วยโปรแกรม TouchDesigner หรืองาน Event ต่างๆ

2. ข้อมูลจำเพาะของฮาร์ดแวร์ (Hardware Specifications & Pinout)
--------------------------------------------------------------------------------
- บอร์ดควบคุมหลัก: WT32-ETH01 (ESP32-D0WDQ6 V3)
- การเชื่อมต่อเครือข่าย: LAN8720A Ethernet PHY (10/100 Mbps)
- การตั้งค่า Ethernet ในโค้ด:
  * PHY Address: 1
  * PHY Power Pin: 16 (หรือไม่ได้เชื่อมต่อ ขึ้นอยู่กับเวอร์ชันบอร์ด แต่ภายในบอร์ดจะต่ออยู่กับ GPIO16)
  * PHY MDC Pin: 23
  * PHY MDIO Pin: 18
  * Clock Mode: ETH_CLOCK_GPIO0_IN
- พอร์ตเชื่อมต่อและขาใช้งานที่รองรับ (Available GPIOs):
  * Output Pins (สำหรับ DMX/LED): GPIO 2, 4, 12, 14, 15, 17, 32, 33 (โดยบอร์ดปัจจุบันที่ทำมาแล้วจะเชื่อมต่อเอาต์พุตเข้ากับ GPIO 4, 12, 14, 15 เป็นหลัก)
  * Input Only Pins: GPIO 35, 36, 39 (ใช้รับเซนเซอร์หรือปุ่มกด ไม่สามารถใช้ส่งสัญญาณได้)
  * ข้อควรระวังเรื่องพินอินพุต/เอาต์พุต (GPIO Constraints):
    - GPIO 4 และ 14: ปลอดภัยสูง ไม่มีเงื่อนไขพิเศษในการบูต สามารถนำมาใช้งานขับ DMX หรือ LED ได้โดยตรง
    - GPIO 12 (MTDI): เป็นพิน Bootstrapping หากพินนี้ถูกดึงค่าเป็น HIGH ตอนเปิดเครื่อง บอร์ดจะคิดว่าต้องอ่านแฟลชที่แรงดัน 1.8V (ซึ่งปกติใช้ 3.3V) ส่งผลให้บอร์ด Boot loop หรือบูตไม่ขึ้น ดังนั้นต้องระวังไม่ให้วงจรฝั่งเอาต์พุต (เช่น ตัวแปลงสัญญาณหรือบอร์ดไดรฟ์เวอร์) ดึงค่าพินนี้เป็น HIGH ตอนกำลังบูตเครื่อง
    - GPIO 15 (MTDO): เป็นพิน Bootstrapping สำหรับควบคุมการพิมพ์ Log ตอนบูต มีตัวต้านทาน Pull-up ภายใน โดยทั่วไปแล้วสามารถนำมาใช้งานเป็นเอาต์พุตได้ตามปกติโดยไม่มีผลกระทบต่อระบบหลัก
- วงจรเอาต์พุต (Output Driver Options):
  * ตัวเลือกที่ 1 (DMX512 - TX Only): ใช้ชิป MAX485 หรือ SP485 แปลงสัญญาณ Serial (UART) เป็น RS-485 (Differential DMX512) โดยใช้เพียงขา TX ของ ESP32 ในการส่งข้อมูลออกอย่างเดียว (ไม่ต้องใช้ขา RX ทำให้ขานั้นว่างสำหรับการรับข้อมูลหรือเซนเซอร์อื่นๆ) ขา DE (Driver Enable) และ RE (Receiver Enable) ของ MAX485 สามารถต่อดึงขึ้นแรงดันสูง (VCC) เพื่อให้ทำงานเป็นโหมดส่งสัญญาณออกตลอดเวลาได้เพื่อประหยัดพินของ ESP32
  * ตัวเลือกที่ 2 (Pixel LED): ใช้ชิป Level Shifter (เช่น 74HCT245 หรือ AHCT125) แปลงระดับแรงดันสัญญาณจาก 3.3V เป็น 5V เพื่อส่งให้ไฟเส้น WS2812B / UCS1903 / SK6812
  * ข้อแนะนำด้านความปลอดภัยระบบไฟ (Power & Safety Guide):
    - ในการต่อระบบไฟของ Pixel LED ที่ดึงกระแสสูง ควรต่อกระบอกฟิวส์ (Fuse) ขนาดที่เหมาะสมคั่นที่สายไฟบวก (VCC 5V/12V) เสมอก่อนจ่ายเข้าเส้นไฟ เพื่อความปลอดภัยและป้องกันอันตรายหากเกิดการลัดวงจร
    - ควรทำการจ่ายไฟเลี้ยงเพิ่มเติม (Power Injection) ที่ต้นเส้นและท้ายเส้นไฟทุกๆ 150-200 ดวง เพื่อป้องกันแรงดันตกคร่อม (Voltage Drop) ซึ่งเป็นสาเหตุให้ไฟปลายเส้นกระตุก สีเพี้ยน หรือกะพริบสั่นระริก

3. คุณสมบัติของระบบ (Core Features)
--------------------------------------------------------------------------------
- [ ] Network & Wi-Fi Configuration Access (การเข้าถึงเพื่อตั้งค่าเครือข่าย):
  * Ethernet Mode: รองรับการรับ IP อัตโนมัติ (DHCP) และการตั้งค่า IP แบบคงที่ (Static IP) ผ่านสาย LAN
  * Wi-Fi SoftAP Mode (โหมดปล่อยสัญญาณ Wi-Fi สำหรับตั้งค่า): เพื่อป้องกันการทับซ้อนกับโหมดไร้สายอื่นๆ จะใช้ตรรกะการทำงานอัจฉริยะดังนี้:
    - โหมดที่ต้องใช้ LAN (Art-Net Ethernet / ESP-NOW Master): หากบูตเครื่องแล้วไม่พบสาย LAN (No Link Status) บอร์ดจะเปิด Wi-Fi SoftAP ทันทีเพื่อให้เข้าไปตั้งค่าได้
    - โหมดไร้สาย (ESP-NOW Slave): เนื่องจากปกติไม่ได้ต่อสาย LAN บอร์ดจะทำงานในโหมดรับสัญญาณ ESP-NOW ควบคู่กับปล่อย Wi-Fi SoftAP ไปพร้อมกันในช่วงแรก (โดย Wi-Fi SoftAP จะปิดตัวเองอัตโนมัติภายใน 2 นาทีหลังบูตหากไม่มีผู้เชื่อมต่อ เพื่อลดการรบกวนของคลื่นสัญญาณในงานจริง) หรือผู้ใช้สามารถเสียบสาย LAN ชั่วคราวเพื่อบังคับให้บอร์ดเข้าโหมดตั้งค่าได้ทันที
  * mDNS Support: รองรับการเข้าหน้าเว็บตั้งค่าผ่านโดเมน `http://artnet.local` โดยไม่ต้องจำ IP (ใช้ได้กับทั้งโหมดต่อสาย LAN และโหมดปล่อย Wi-Fi ในตัว)
  * Diagnostic Status LED (ไฟแสดงสถานะการวินิจฉัย - Onboard Serial LEDs Hack): เนื่องจากโหมด DMX ทำงานแบบ TX-Only (ส่งข้อมูลออกทางเดียว) ทำให้พิน RXD2 (GPIO 5) ว่างและไม่มีการเชื่อมต่อภายนอก เราจึงสามารถใช้ไฟแสดงสถานะดั้งเดิมบนบอร์ดได้ทันทีโดยไม่ต้องต่อไฟภายนอกเพิ่ม:
    - LED3 (GPIO 5 - พิน RXD2 เดิม): กำหนดพินนี้เป็น Digital Output ในโค้ดเพื่อควบคุมการเปิด-ปิดไฟ LED3 บนบอร์ดโดยตรง (กะพริบช้าเมื่ออยู่โหมดตั้งค่า SoftAP / ดับหรือสว่างค้างเมื่อสถานะปกติ)
    - LED4 (GPIO 17 - พิน TXD2 เดิม): เมื่อใช้พินนี้ส่งสัญญาณ DMX512 สัญญาณข้อมูลระดับฮาร์ดแวร์จะทำให้ไฟ LED4 กะพริบถี่ๆ แสดงความเคลื่อนไหวข้อมูลแสงสว่างโดยอัตโนมัติ
    - ทั้งนี้ ระบบยังรองรับการเลือกเปลี่ยนพินไฟแสดงสถานะนี้ไปยัง GPIO อื่นๆ สำหรับต่อไฟภายนอกได้ผ่านหน้าเว็บคอนฟิก
- [ ] Art-Net Protocol Support:
  * รองรับโปรโตคอล Art-Net 3 หรือ Art-Net 4 (รับส่งข้อมูลแบบ UDP Port 6454)
  * รองรับฟังก์ชัน ArtPoll และตอบกลับด้วย ArtPollReply เพื่อให้อุปกรณ์แสดงผลบนซอฟต์แวร์ควบคุมและพัฒนาคอนเทนต์โดยอัตโนมัติ (เช่น Resolume Arena, TouchDesigner, Madrix, QLC+, MA2/MA3)
  * รองรับการรับข้อมูลแบบทั้ง Broadcast และ Unicast (การส่งแบบ Unicast แนะนำสำหรับโปรเจกต์ขนาดใหญ่เพื่อลดภาระ CPU บน ESP32)
- [ ] Output Drivers (DMX TX-Only / LED):
  * โหมด DMX512: ส่งข้อมูล DMX ออกทาง Hardware Serial (ใช้เฉพาะขา TX) ด้วยความเร็ว 250Kbps พร้อมความสามารถในการปรับอัตราเฟรมส่งออก (DMX Frame Rate / FPS) ในหน้าเว็บ เพื่อป้องกันการกะพริบของโคมไฟราคาถูกที่ทนอัตราสัญญาณที่ส่งเร็วเกินไปไม่ได้
  * โหมด Pixel LED & Multi-Universe Mapping: ขับไฟเส้นแบบ Addressable LED ด้วยไลบรารีประสิทธิภาพสูง (FastLED/NeoPixelBus) พร้อมระบบดึงข้อมูลสลับข้ามหน้าจักรวาลอัตโนมัติ (170 พิกเซล หรือ 510 ช่อง DMX ต่อ 1 Universe) เพื่อให้สามารถขับไฟเส้นจำนวนมากข้ามจักรวาลได้อย่างสมบูรณ์แบบโดยแสดงผลต่อกันอย่างมีระเบียบ
- [ ] ESP-NOW Wireless Bridge Mode (เชื่อมโยงสัญญาณไร้สายระหว่าง ESP32 ไม่ต้องใช้เราเตอร์):
  * รองรับกรณีเดินสาย LAN ลำบากและไม่ต้องการติดตั้งเราเตอร์ไร้สาย (Router-less Multi-Node Setup)
  * โหมด Master (Transmitter): รับข้อมูล Art-Net จาก Ethernet LAN แล้วแพร่สัญญาณ DMX แบบไร้สายผ่านโปรโตคอล ESP-NOW
  * โหมด Slave (Receiver): รับสัญญาณ DMX จากตัว Master ผ่าน ESP-NOW ไร้สาย แล้วนำไปส่งออกทาง DMX TX หรือ Pixel LED
- [ ] WiZ Smart Bulb Control Mode (โหมดควบคุมหลอดไฟ WiZ ไร้สาย):
  * สามารถรับสัญญาณ Art-Net แล้วแปลงไปควบคุมหลอดไฟ WiZ Wi-Fi (รุ่น 8.5W) ได้โดยตรงผ่านเครือข่ายวง LAN/Wi-Fi เดียวกัน
  * ใช้โปรโตคอลน้ำหนักเบา WiZ Local UDP (ส่งคำสั่ง JSON string ไปยังพอร์ต 38899 ของหลอดไฟ) เพื่อปรับระดับสีและแสงสว่าง
  * ระบบมีกลไกจำกัดอัตราส่งข้อมูล (Rate Limiting) อยู่ที่ 5-10 ครั้ง/วินาที (5-10 Hz) เพื่อป้องกันไมโครคอนโทรลเลอร์ขนาดเล็กของหลอดไฟ WiZ แฮงก์หรือหลุด Wi-Fi
  * รองรับโหมดจัดสรรช่องสัญญาณ DMX (DMX Mapping Modes) สองรูปแบบเพื่อให้ง่ายต่อการควบคุมด้วยโปรแกรมอื่นๆ:
    - โหมด 3 ช่อง (RGB Only Mode): ใช้ DMX เพียง 3 ช่องสำหรับควบคุมสี R, G, B (โดยเฟิร์มแวร์ในบอร์ด ESP32 จะคำนวณหาค่าความสว่าง Dimming จากค่าสูงสุดของ R, G, B แล้วทำการปรับสเกลแม่สีส่งให้หลอด WiZ อัตโนมัติ ทำให้ผู้ใช้จับคู่โคมไฟเป็น Generic RGB ทั่วไปในโปรแกรมคุมไฟได้ทันที)
    - โหมด 4 ช่อง (RGB + Dimmer Mode): ใช้ DMX 4 ช่องแยกกันคุมสี R, G, B และ Dimmer อิสระ
  * ข้อจำกัดจำนวนหลอดไฟ (Bulb Count Limits & Best Practices):
    - แบบคุมเป็นกลุ่ม (Broadcast Mode): ใช้ IP ปลายทางเป็นบรอดแคสต์ (เช่น `255.255.255.255`) สั่งงาน**ไม่จำกัดจำนวนหลอด** (กี่ร้อยหลอดก็ได้) โดยใช้สัญญาณเพียง 1 แพ็กเก็ต ทำให้ไม่มีปัญหากับเครือข่าย (ไฟทุกหลอดเปลี่ยนสีพร้อมกันหมด)
    - แบบคุมแยกอิสระ (Unicast Mode - แยกกันคนละสีคนละช่อง DMX): แนะนำจำกัดไม่เกิน **8 - 16 หลอดต่อบอร์ด** เนื่องจากบอร์ดต้องส่ง UDP แยกไปทีละ IP ซึ่งหากมีหลอดไฟมากเกินไป จะทำให้เครือข่าย Wi-Fi หนาแน่น และเกิดความหน่วงสูงขึ้น
  * สามารถจับคู่ช่องสัญญาณ DMX และระบุ IP ของหลอดไฟ WiZ (เดี่ยว/กลุ่ม) ผ่านหน้าเว็บตั้งค่าได้โดยตรง
- [ ] Standalone Mode & Sequence Builder (โหมดอัตโนมัติและการสร้างโปรแกรมไฟด้วยตนเอง):
  * ทำงานอัตโนมัติเมื่อขาดการเชื่อมต่อเครือข่าย หรือไม่มีสัญญาณ Art-Net เข้ามาตามเวลาที่กำหนด (เช่น 5 วินาที)
  * Step-based Sequence Builder: ให้ผู้ใช้สามารถออกแบบสเต็ปไฟเองได้ผ่านหน้าเว็บ (เช่น สเต็ปที่ 1: สีแดง สว่าง 100% เฟด 1 วิ ค้าง 2 วิ -> สเต็ปที่ 2: สีน้ำเงิน...) โดยตัวควบคุมจะบันทึกสเต็ปเหล่านี้เป็น JSON ลง Flash Memory (LittleFS)
  * Built-in Parameterized Effects: มีโปรแกรมไฟสำเร็จรูปในตัว (เช่น Rainbow, Strobe, Color Fade, Fire) ที่สามารถปรับความเร็ว ความสว่าง และโทนสีได้จากหน้าเว็บ
  * Fail-safe Mode (ตรรกะเมื่อเน็ตเวิร์กหลุด): เลือกตั้งค่าพฤติกรรมของไฟได้หากตรวจพบว่าสัญญาณ Art-Net ขาดหายไป (เช่น ค้างสถานะไฟล่าสุดไว้ Hold State, ดับไฟทั้งหมดทันที Blackout, หรือเปลี่ยนเป็นเล่นโปรแกรมไฟ Standalone อัตโนมัติ)
  * NTP Time Sync & Scheduled Standalone Tasks: ซิงค์เวลากับอินเทอร์เน็ตผ่าน NTP เมื่อเชื่อมต่อสาย LAN ทำให้ตั้งเวลาเปิด-ปิดไฟ หรือเปลี่ยนโปรแกรมไฟ Standalone ตามเวลาจริงของวันได้อัตโนมัติ (เช่น สั่งเปิดไฟวิ่งเบาๆ ตั้งแต่ 18:00 น. และปิดไฟระบบตอน 02:00 น. ทุกวัน) เหมาะสำหรับงานบาร์หรือร้านอาหารที่ต้องการการทำงานอัตโนมัติแบบไร้รอยต่อ
  * ข้อจำกัดและแนวทางแก้ไข (Limitations & Solutions):
    1. ข้อจำกัดหน่วยความจำ: ไม่รองรับการอัดสัญญาณ DMX ดิบ (Raw DMX Recording) เนื่องจากใช้พื้นที่ Flash สูงเกินไป $\rightarrow$ แก้ไขโดยใช้ Step-based (เก็บค่าสี เวลา และเอฟเฟกต์ในรูป JSON) ซึ่งใช้พื้นที่น้อยมาก
    2. ข้อจำกัดประสิทธิภาพ (ระบบประมวลผลขนาน Dual-Core CPU): บอร์ด ESP32-ETH01 ไม่มี GPU (ชิปการ์ดจอ) แต่มี **Dual-Core CPU (2 คอร์หลัก)** ความเร็ว 240MHz $\rightarrow$ แก้ไขปัญหาประสิทธิภาพโดยใช้ FreeRTOS แยกหน้าที่การทำงานของแต่ละคอร์ออกจากกันอย่างเด็ดขาด:
       - **Core 0 (คอร์เน็ตเวิร์ก):** จัดการเฉพาะระบบเชื่อมต่อเครือข่าย (Ethernet/TCP/IP stack, การรับแพ็กเก็ต Art-Net UDP, หน้าเว็บ Config Server) เพื่อให้รับส่งข้อมูลได้รวดเร็วและไม่หลุดการเชื่อมต่อ
       - **Core 1 (คอร์การแสดงผล):** คำนวณเอฟเฟกต์โปรแกรมไฟ Standalone และขับสัญญาณเอาต์พุต DMX / Pixel LED (ขับสัญญาณร่วมกับฮาร์ดแวร์ DMA/RMT) ทำให้สัญญาณไฟนิ่ง ไม่กะพริบกระตุก แม้เน็ตเวิร์กจะมีข้อมูลหนาแน่น
    3. ข้อจำกัดความง่ายในการใช้งาน: การเขียนโค้ดสคริปต์บนหน้าเว็บทำได้ยาก $\rightarrow$ แก้ไขโดยออกแบบเป็น Web Visual Builder (ช่องเลือกสี Color Picker, สไลเดอร์ปรับความเร็ว/ความสว่าง, ปุ่มกดเพิ่ม/ลบสเต็ป) ใช้งานง่ายบนมือถือและ PC
- [ ] Web Configuration Portal:
  * มีหน้าเว็บภายในตัว (Web Server) สำหรับตั้งค่าอุปกรณ์ผ่าน Web Browser
  * รองรับ Responsive Design สามารถแสดงผลและตั้งค่าได้อย่างสมบูรณ์แบบทั้งบนมือถือ (Mobile) และคอมพิวเตอร์ (PC)
  * การแสดงผลโลโก้แบบประหยัดหน่วยความจำขั้นสูงสุด (Ultra-low Memory Logo):
    - สำหรับไฟล์โลโก้ `[CHAL]_128_64.bmp` (ขนาด 128x64 พิกเซล) จะแปลงให้เป็น 1-bit PNG ขนาดเล็กมาก (บีบอัดเหลือ 266 ไบต์)
    - นำรหัสรูปภาพแปลงเป็น Base64 แล้วฝังไว้ในหน้า HTML โดยตรง (Inline Data URI ขนาด 356 ตัวอักษร) ทำให้ประหยัดแรม (RAM) บน ESP32 เป็น 0 ไบต์ และไม่ต้องทำไฟล์รูปแยกในระบบไฟล์ ทำให้หน้าเว็บโหลดเร็วที่สุดและไม่กินสเปกเครื่อง
  * ค่าที่ตั้งได้: IP Address, Netmask, Gateway, Universe, Device Mode (Artnet Ethernet / ESP-NOW Master / ESP-NOW Slave / Standalone), เลือกเอฟเฟกต์ Standalone, ประเภทของเอาต์พุต (DMX หรือ LED), การตั้งค่าขาพินควบคุม (DMX TX Pin, LED Data Pin, Status LED Pin), จำนวน LED, การจัดเรียงสี (RGB, GRB ฯลฯ)
  * บันทึกค่าการตั้งค่าลงใน Flash Memory (Preferences) เพื่อให้ค่าไม่หายเมื่อดับไฟ โดยโปรแกรมหลักจะโหลดพินและค่าคอนฟิกไปเริ่มต้นระบบแบบไดนามิก ทำให้ผู้ใช้งานปรับเปลี่ยนพอร์ตเชื่อมต่อได้ภายหลังโดยไม่ต้องแก้โค้ดและแฟลชใหม่
- [ ] Over-The-Air (OTA) Update:
  * สามารถอัปเกรดเฟิร์มแวร์ผ่านหน้าเว็บได้โดยไม่ต้องถอดบอร์ดออกมาต่อสาย USB

4. แผนการดำเนินงานและขั้นตอนพัฒนา (Development Stages)
--------------------------------------------------------------------------------

ระยะที่ 1: การเตรียมการและการออกแบบฮาร์ดแวร์ (Hardware Preparation)
  * [ ] จัดหาบอร์ด WT32-ETH01, บอร์ดแปลงสัญญาณ MAX485, ชิป Level Shifter และไฟเส้นทดสอบ
  * [ ] ออกแบบวงจรและการเชื่อมต่อระหว่าง WT32-ETH01 กับอุปกรณ์เอาต์พุต
  * [ ] ประกอบวงจรบน Breadboard สำหรับใช้ทดสอบเบื้องต้น

ระยะที่ 2: ตั้งค่าระบบพัฒนาและสร้างโครงสร้างโปรเจค (Environment Setup)
  * [ ] ติดตั้ง PlatformIO (แนะนำเพื่อความเป็นระเบียบและการจัดการไลบรารีที่ง่าย)
  * [ ] สร้างโปรเจคสำหรับ WT32-ETH01 และระบุค่าบอร์ดใน `platformio.ini`
  * [ ] เพิ่มและทดสอบโค้ดเริ่มต้นการทำงานของ Ethernet (LAN8720) เพื่อให้แน่ใจว่าเชื่อมต่อสาย LAN และได้รับ IP

ระยะที่ 3: พัฒนาส่วนควบคุมสัญญาณเครือข่ายและการรับส่งข้อมูล (Network & Wireless Communication Core)
  * [ ] เขียนฟังก์ชันการรับแพ็กเก็ต UDP พอร์ต 6454 (Art-Net) ผ่าน Ethernet
  * [ ] เขียนระบบรับ-ส่งข้อมูล DMX ไร้สายระหว่างตัวอุปกรณ์ ESP32 ผ่านโปรโตคอล ESP-NOW
  * [ ] เขียนระบบเปิด Wi-Fi SoftAP และการจัดการโหมดการทำงานอัจฉริยะ (Smart Configuration Logic) เพื่อแยกแยะความต้องการใช้เน็ตเวิร์กของแต่ละโหมด:
    - สำหรับโหมด Ethernet/Master: ตรวจไม่พบสาย LAN -> เปิด SoftAP ทันที
    - สำหรับโหมด ESP-NOW Slave: เปิดใช้งาน ESP-NOW พร้อมปล่อย Wi-Fi SoftAP แบบกำหนดเวลาปิด (SoftAP Timeout) เพื่อป้องการสัญญาณกวนกันขณะใช้งานจริง และเขียนคำสั่งเปิดใช้ SoftAP เมื่อมีการตรวจพบการเสียบสาย LAN เพื่อใช้ตั้งค่าฉุกเฉิน
  * [ ] เขียนฟังก์ชันแปลงสัญญาณจาก ArtDMX ไปยังช่องสัญญาณ DMX ทั้งแบบมีสายและไร้สาย
  * [ ] เขียนระบบรองรับ ArtPollReply เพื่อแสดงตัวตนในระบบเครือข่าย
  * [ ] ทดสอบอัตราการสูญหายของเฟรมและสัญญาณหน่วง (Latency Test) ทั้งโหมดสายและไร้สาย ESP-NOW

ระยะที่ 4: พัฒนาส่วนควบคุมเอาต์พุตและเอฟเฟกต์อัตโนมัติ (Output Drivers & Standalone Engine)
  * [ ] กรณี DMX: เขียนโค้ดส่งข้อมูล Serial 250kbps แบบ TX-Only พร้อม Break Signal & MAB ตามมาตรฐาน DMX512
  * [ ] กรณี LED: พัฒนาโค้ดส่งสัญญาณควบคุมไฟเส้นด้วยพอร์ต RMT/I2S ของ ESP32 เพื่อหลีกเลี่ยงผลกระทบจากการประมวลผลเครือข่าย
  * [ ] พัฒนาเอฟเฟกต์โปรแกรมไฟพื้นฐาน (เช่น Rainbow, Fade, Solid Color, Strobe) สำหรับการทำงานอิสระ (Standalone) ในตัวชิป
  * [ ] เขียนกลไกส่งคำสั่งผ่าน WiZ Local UDP (พอร์ต 38899) แปลงค่า DMX เป็นคำสั่ง JSON พร้อมระบบจำกัดอัตราการส่ง (Rate Limiting) ที่เหมาะสม
  * [ ] เขียนกลไกการสลับโหมดอัตโนมัติเมื่อเครือข่ายหลุด (Fail-safe auto switch to Standalone)
  * [ ] ทดสอบความเสถียรในการทำงานและการกะพริบของไฟ (Flicker & Stability Test)

ระยะที่ 5: พัฒนาเว็บสำหรับตั้งค่าและบันทึกค่า (Web Config & Preferences)
  * [ ] สร้างหน้าเว็บ HTML/CSS/JS แบบ Responsive Design (ใช้ Media Queries, Flexbox/Grid) ปรับเลย์เอาต์ตามขนาดหน้าจอมือถือและ PC อัตโนมัติ เพื่อให้ใช้ง่านง่ายทุกหน้าจอ
  * [ ] เพิ่มตัวเลือกในหน้าเว็บสำหรับการสลับโหมดอุปกรณ์ (Artnet-Ethernet / ESP-NOW Master / ESP-NOW Slave / Standalone)
  * [ ] เพิ่มตัวเลือกในหน้าเว็บสำหรับตั้งค่า DMX Refresh Rate, จำนวนพิกเซล LED, ลำดับสีของไฟเส้น (เช่น GRB, RGB) และจักรวาลเริ่มต้น (Starting Universe) เพื่อรองรับการแมปข้ามจักรวาลอัตโนมัติ
  * [ ] เพิ่มช่องกรอกข้อมูลตั้งค่าสำหรับการควบคุมหลอดไฟ WiZ (ระบุ IP ของหลอดไฟ, DMX Start Channel และเลือกโหมดการแมปสัญญาณ DMX แบบ RGB 3 ช่อง หรือ RGB+Dimmer 4 ช่อง)
  * [ ] เพิ่มหน้าเว็บควบคุมเลือกโปรแกรมไฟ Standalone และความเร็ว/ความสว่างแบบเรียลไทม์
  * [ ] เขียนโค้ด REST API บน ESP32 เพื่อรับ-ส่งข้อมูลการตั้งค่าในรูปแบบ JSON และผูกเข้ากับระบบ Preferences ใน Flash Memory
  * [ ] เพิ่มระบบอัปโหลดเฟิร์มแวร์ผ่าน OTA Web Update

ระยะที่ 6: การทดสอบระบบโดยรวมและการทำโปรดักชั่น (System Integration & Assembly)
  * [ ] ทดสอบการทำงานต่อเนื่องระยะยาว (Stress Test) ตรวจสอบปัญหา Memory Leak หรือการหยุดทำงานของเครือข่าย
  * [ ] ทดสอบร่วมกับซอฟต์แวร์ควบคุมแสงและโปรแกรมสร้างอินเตอร์แอคทีฟหลัก เช่น TouchDesigner (ผ่าน DMX Out CHOP), Resolume Arena, QLC+
  * [ ] ออกแบบและกัดแผ่นปริ้นต์ (PCB Shield) สำหรับ WT32-ETH01 เพื่อเชื่อมสายไฟอย่างมั่นคง
  * [ ] ออกแบบและพิมพ์กล่องสำหรับใส่อุปกรณ์ด้วยเครื่องพิมพ์ 3 มิติ (3D Print Case)

5. รายการไลบรารีและเครื่องมือที่ใช้ (Proposed Libraries)
--------------------------------------------------------------------------------
- IDE/Framework: PlatformIO / Arduino Framework
- Ethernet: `ETH.h` (ไลบรารีมาตรฐานของ ESP32 Arduino Core)
- Async Web Server: `ESPAsyncWebServer` & `AsyncTCP` (สำหรับหน้าเว็บตั้งค่าที่ลื่นไหล)
- JSON Parser: `ArduinoJson` (สำหรับรับส่งค่าคอนฟิก)
- LED Driver: `FastLED` หรือ `NeoPixelBus` (NeoPixelBus จะมีความเสถียรสูงบน ESP32 เมื่อใช้งานร่วมกับ Wi-Fi/Ethernet)
- DMX Driver: `esp_dmx` หรือเขียนการขับ UART โดยตรงผ่าน ESP32 UART Driver
